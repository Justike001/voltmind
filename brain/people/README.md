# People

One page per person relevant to work.

Use this for:
- Employees
- Contractors
- Advisors
- Candidates
- Partner/customer contacts
- Internal or external stakeholders

Required frontmatter includes:
- `email`
- `work_location`
- `job_title`
- `department`
- `team`
- `manager`
- `employment_status`
- `aliases`

Page purpose:
- Who this person is in the operating context
- What they own
- What they know
- Current work links
- Open threads linked to state objects

Do not use this for speculative personality profiles.
